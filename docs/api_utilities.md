# API Utilities — Быстрые утилиты для Ozon и WB

**Путь:** `C:\Users\Admin\`  
**Язык:** Python 3.10+  
**Запуск:** `uv run <script>.py`  
**Статус:** Разовые утилиты

---

## Что это

Три отдельных скрипта в корне пользователя — быстрые однофайловые утилиты для проверки данных через API Ozon и Wildberries. Не часть какого-то проекта, просто удобные инструменты под рукой.

---

## Скрипты

### `check_ozon_stocks.py` — остатки Ozon

**Что делает:**
1. Запрашивает список первых 10 товаров через `POST /v2/product/list`
2. Берёт `offer_id` первых 5 товаров
3. Проверяет остатки через `POST /v4/product/info/stocks`
4. Выводит результат в консоль

**Настройки (жёстко вшиты в файл):**
```python
CLIENT_ID = "3315303"
API_KEY   = "6ee3b7b7-eef3-4ce5-ac54-0b830613c55a"
```

**Запуск:**
```bash
cd C:\Users\Admin
uv run check_ozon_stocks.py
```

---

### `fetch_wb_commissions.py` — комиссии Wildberries

**Что делает:**  
Получает текущие комиссии WB по категориям через WB API.  
Выводит в консоль таблицу: категория → % комиссии.

**Запуск:**
```bash
cd C:\Users\Admin
uv run fetch_wb_commissions.py
```

---

### `fetch_wb_tariffs.py` — тарифы Wildberries

**Что делает:**  
Получает тарифы WB на логистику: хранение, приёмку, доставку.  
Полезно для расчёта unit-экономики.

**Запуск:**
```bash
cd C:\Users\Admin
uv run fetch_wb_tariffs.py
```

---

## Когда использовать

- Быстро проверить что с остатками на Ozon
- Уточнить комиссию WB для новой категории
- Посмотреть тарифы WB для расчёта цены

Для постоянной работы лучше использовать скрипты из `Autoparts_Ecommerce` или `Ecommerce` — они сохраняют данные и ведут историю.
