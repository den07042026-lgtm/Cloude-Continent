## Блок Яндекс.Диска (фотографии)

**Статус:** заготовлен, ожидает загрузки фото на Яндекс.Диск.

**Логика:**
- Все фото в **одной публичной папке** на Яндекс.Диске.
- Поиск фото по **вхождению артикула в имя файла**.
  - Пример: артикул `a22025` → ищем файлы содержащие `a22025` в имени.
- Файл с окончанием `_info` → **главное фото** (col 15).
  - Пример: `f-a22025_1_info.jpg`
- Файлы с окончанием `_compat` → **дополнительные фото** (col 16, через `\n`).
  - Пример: `f-a22025_1_compat.jpg`, `f-a22025_2_compat.jpg`

**Как активировать** — одна строка в начале скрипта (строка 47):
```python
YADISK_FOLDER_URL = "https://disk.yandex.ru/d/XXXXXXXXXXXXXXX"
```

**Как получает ссылки:** через публичный API Яндекс.Диска без OAuth:
`https://cloud-api.yandex.net/v1/disk/public/resources?public_key=<URL>`
Поле `file` в ответе — прямая ссылка на скачивание (действует несколько часов).
Шаблон нужно загрузить в Ozon сразу после запуска скрипта.

## Ключевые решения (зафиксировать для ИИ)

- **НДС** = всегда `Не облагается`
- **Цена до скидки** = не заполняется
- **ТН ВЭД** = не заполняется
- **Страна-изготовитель** = не заполняется (нет данных в источнике)
- **Партномер** = Код Mikado БЕЗ суффикса (оригинальный артикул производителя)
- **Артикул** = Код Mikado + `-con` (уникальная система артикулов продавца)
- **Ускоренный сбор отзывов** = `Да` для всех товаров
- **Доп. фото** — через перенос строки `\n` (требование Ozon для модерации)
