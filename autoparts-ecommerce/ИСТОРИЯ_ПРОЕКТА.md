## 1. Суть проекта

Продажа автозапчастей на Ozon по модели FBS. Основной поставщик — Микадо (Волгоград, склад ID=34).  
Задача — полная автоматизация: синхронизация остатков, автоматические заказы у поставщика, ежедневный пересчёт цен.

---

## 2. Архитектура системы

```
Ozon → МойСклад (авто-интеграция) → ozon_order_sync.py → Микадо + Telegram
Mikado price (каждые 2ч) → ozon_stock_sync.py → МойСклад → Ozon
Mikado price (00:01 ежедневно) → price_recalc.py → МойСклад цены → Ozon
```

**МойСклад** — центральный хаб: Ozon автоматически синхронизирует заказы и остатки через встроенную интеграцию.

### `scripts/ozon_order_sync.py` *(полная переработка)*
Читает новые заказы из МойСклад, заказывает у Микадо, шлёт Telegram.

**Опрос:** каждые 15 минут.  
**State-файл:** `data/order_sync_state.json` — хранит обработанные ID и последний timestamp.

**Процесс заказа у Микадо (реальная логика сайта):**

1. POST `galleyp.asp` с полем `CODE=<код>` → редирект на `SearchCodeG.asp?CODE=<код>`
2. На странице результатов берём **первую** ссылку вида `href='galleyp.asp?code=...'`  
   *(первый результат = нужный нам номер, остальные — аналоги)*
3. GET карточки товара `galleyp.asp?code=xzk-of-4063`
4. Если строки **«Волгоград»** нет в HTML → товара нет на складе → Telegram алерт  
   *(нулевые остатки не показываются, строка просто отсутствует)*
5. Из скрытого поля `MaxQTY` берём количество в Волгограде
6. POST `pp0.asp?MODE=AddOrd&R=<timestamp>` со всеми полями формы `formADD`:
   - `VOLUME` = количество к заказу
   - `COMMAND` = ADD
   - `CODE` = полный код с префиксом бренда (напр. `xzk-of-4063`)
   - `INSERT` = Заказать
   - `StockID` = 34 (склад Волгоград)
   - `ExprList`, `EXPR`, `OEMID`, `json`, `MaxQTY` — берутся из скрытых полей страницы
7. В ответе `pp0.asp` при успехе приходит `Результат: <b>OK</b>`

**Ключевые константы:**
```python
MIKADO_LOGIN_URL  = "https://mikado-parts.ru/office/SECURE.asp"
MIKADO_SEARCH_URL = "https://mikado-parts.ru/office/galleyp.asp"
MIKADO_ORDER_URL  = "https://mikado-parts.ru/office/pp0.asp"
ARTICLE_SUFFIX    = "-con"   # "a22025-con" → "a22025" (код Микадо)
POLL_INTERVAL_MINUTES = 15
```

**Авторизация Микадо:**
```python
session.post(MIKADO_LOGIN_URL, data={"CODE": "35275", "PASSWORD": "[REDACTED]", "INSERT": "OK"})
# Успех: в HTML есть "выход", "Продолжить" или "Обслуживание клиентов"
```

### `scripts/price_recalc.py` *(новый)*
Ежедневно в 00:01 пересчитывает цены по формуле 12% маржи FBS → обновляет МойСклад → Ozon.

**Формула ценообразования (FBS, цель 12% маржи):**
```python
TARGET_MARGIN = 0.12
ACQ_PCT  = 0.015   # эквайринг 1.5%
TAX_PCT  = 0.06    # УСН 6%
RET_RATE = 0.03    # возвраты 3%
REVERSE  = 80      # обратная логистика 80₽
OTHER    = 30      # прочие расходы 30₽

# Комиссия FBS по ценовым тирам:
FBS_TIERS = [100, 300, 1500, 5000, 10000]
FBS_RATES = [0.14, 0.20, 0.44, 0.44, 0.44, 0.44]

# Логистика FBS (по расчётному весу):
# расчётный вес = max(фактический кг, объёмный кг)
# объёмный кг = Д×Ш×В (мм) / 5 000 000
LOG_FBS = [(0.5,75),(1,90),(2,115),(5,155),(10,210),(15,265),(20,315),(25,365),(30,420),(50,620)]
DEFAULT_LOGISTICS = 115  # ₽, если нет данных о габаритах (≈2 кг)

# Прибыль:
profit = sell - purchase - commission - acquiring - logistics - return_loss - other - tax
# return_loss = sell*ret_rate*(1 - fbs_rate) + reverse*ret_rate
# tax = sell * TAX_PCT
margin = profit / sell  # должно быть >= 0.12

# Поиск цены: линейный перебор 50..500000₽ с шагом 1₽
```

**МойСклад: цены хранятся в копейках (× 100).**  
Обновление: `PUT /entity/product/{id}` с полем `salePrices`.

## 5. API МойСклад

**Base URL:** `https://api.moysklad.ru/api/remap/1.2`  
**Auth:** `Authorization: Bearer <MOYSKLAD_TOKEN>`

| Задача | Метод | Endpoint |
|--------|-------|----------|
| Получить заказы | GET | `/entity/customerorder?filter=moment>TIMESTAMP&expand=positions,positions.assortment` |
| Обновить остатки | POST | `/entity/inventory` |
| Список товаров | GET | `/entity/product?limit=100&offset=N` |
| Обновить цену | PUT | `/entity/product/{id}` |
| Получить организацию | GET | `/entity/organization` |
| Получить склад | GET | `/entity/store` |
| Тип цены | GET | `/entity/pricetype` → ищем «Цена продажи» |
| Валюта | GET | `/entity/currency` → ищем код «RUB» |

**Артикул в МойСклад** хранится в поле `assortment.article` формата `a22025-con`.  
Код Микадо = артикул без суффикса `-con`: `"a22025-con" → "a22025"`.
