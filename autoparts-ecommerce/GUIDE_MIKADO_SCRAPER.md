## СУТЬ ПРОЕКТА

Система автоматического сбора данных о товарах с сайта поставщика автозапчастей **mikado-parts.ru**.
Читает артикулы из Excel прайса, авторизуется на сайте, собирает все данные по каждому товару,
сводит в Excel таблицу и скачивает фотографии.

## КАК ЗАПУСТИТЬ

### Стандартный запуск (строки 2-20 прайса):
```
cd C:\Users\Admin\Documents\Autoparts_Ecommerce
uv run --with requests,openpyxl scripts/mikado_scraper.py --price "C:/Users/Admin/Documents/Ecommerce/mikado_price_34.xlsx" --rows 2-20
```

### Все аргументы скрипта:
| Аргумент | Описание | Пример |
|---|---|---|
| `--price` | Путь к Excel прайсу (обязательный) | `--price "C:/путь/прайс.xlsx"` |
| `--rows` | Диапазон строк прайса | `--rows 2-50` |
| `--out` | Папка для результатов | `--out "C:/Desktop/output"` |
| `--login` | Код клиента (если нет .env) | `--login 35275` |
| `--password` | Пароль (если нет .env) | `--password XXXXX` |
| `--delay` | Пауза между запросами, сек | `--delay 2.0` |
| `--env` | Путь к .env файлу | `--env "C:/path/.env"` |

## ЧТО СОБИРАЕТ СКРИПТ

По каждому артикулу из прайса скрипт делает 5 запросов к сайту:

| Запрос | URL | Что получает |
|---|---|---|
| Основная карточка | `galleyp.asp?code=f-a22025` | название, бренд, цена, склады, параметры, ARTID |
| OEM номера | `pp0.asp?MODE=OEM&ARTID=...` | список производитель + OEM код |
| Применяемость | `pp0.asp?MODE=APL&ARTID=...` | марка / модель / годы выпуска |
| Изображения | `pp0.asp?MODE=PIC&ARTID=...` | JPEG файлы |
| Перекодировки | `pp0.asp?Mode=Cross&code=...` | альтернативные артикулы |

## КАК РАБОТАЕТ АВТОРИЗАЦИЯ

Сайт mikado-parts.ru использует классический ASP + кодировку windows-1251.
Авторизация — обычный POST запрос, никакого браузера не нужно:

```python
session.post("https://mikado-parts.ru/office/SECURE.asp",
    data={"CODE": "35275", "PASSWORD": "...", "INSERT": "OK"})
```

После этого сессия хранит куки и все последующие запросы проходят как авторизованные.

## ТЕХНИЧЕСКИЕ ДЕТАЛИ ПАРСИНГА

### Прайс (Excel)
- Колонка **Prodnum** — артикул для URL (например `f-a22025`)
- Колонка **Code** — артикул производителя (например `a22025`)
- Скрипт использует Prodnum для запросов к сайту

### Параметры товара
- HTML: `<tbody id=params>` (без кавычек у атрибута — особенность сайта)
- Повторяющиеся ключи объединяются через ` / ` (например "Сторона установки: слева / справа")
- Параметры разные у разных типов товаров: у амортизаторов одни, у ламп другие
- Поэтому используется один столбец "Параметры" вместо отдельных колонок

### Применяемость
- HTML таблица с rowspan: марка и модель занимают несколько строк
- 2 ячейки с rowspan в строке → новая марка + новая модель
- 1 ячейка с rowspan → новая модель (марка остаётся)
- 0 → очередная модификация

### Изображения
- Определяются по magic bytes: JPEG = `FFD8`, PNG = `89504E47`
- Дедупликация по MD5 (один файл может отдаваться по разным URL)
- Именование: `{артикул}_{номер}.jpg`

## ЗАВИСИМОСТИ

Скрипт использует только две внешние библиотеки:
- `requests` — HTTP запросы
- `openpyxl` — работа с Excel

Устанавливаются автоматически при запуске через `uv run --with requests,openpyxl`

**uv** — единственная программа которую нужно иметь установленной на компьютере.

## ТИПИЧНЫЕ ПРОБЛЕМЫ И РЕШЕНИЯ

| Проблема | Причина | Решение |
|---|---|---|
| "Файл не найден" | Неверный путь к прайсу | Проверить путь, возможно файл переименован |
| "Ошибка авторизации" | Неверный логин/пароль | Проверить .env файл |
| PermissionError на xlsx | Файл открыт в Excel | Закрыть файл в Excel |
| Скрипт работает медленно | Нормально | Пауза 1.5 сек между запросами, 19 артикулов ≈ 3 минуты |

## ИСТОРИЯ РАЗРАБОТКИ (кратко)

1. Обнаружили что сайт использует windows-1251 и классический ASP → `requests.Session()` достаточно
2. Параметры парсились неверно → атрибут `id=params` без кавычек, исправили regex
3. OEM и применяемость грузятся через AJAX → нашли эндпоинты в JavaScript страницы
4. Применяемость использует rowspan → написали парсер учитывающий структуру таблицы
5. Изображения дедуплицируются по MD5 (два URL могут вести на один файл)
6. Перекодировки грузятся через `pp0.asp?Mode=Cross` → нашли в JS на странице
7. Параметры сделали единым столбцом вместо отдельных колонок — товары разнотипные
