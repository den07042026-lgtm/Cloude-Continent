## СУТЬ СКРИПТА

`scripts/photo_processor.py` — обработка исходных фото от поставщика mikado-parts.ru.
На каждый товар создаёт **2 фотографии**:
- `_main.jpg` — оригинал без полосы артикула, растянутый по ширине (главное фото Ozon)
- `_info.jpg` — то же фото + логотип магазина сверху-слева + название товара + параметры снизу

## КАК ЗАПУСТИТЬ

```
cd C:\Users\Admin\Documents\Autoparts_Ecommerce
uv run --with pillow --with openpyxl scripts/photo_processor.py --data "data/suppliers/mikado/scraper_output/mikado_data.xlsx" --rows 2-5 --logo "C:/Users/Admin/Documents/Ecommerce/logo.png"
```

### Все аргументы:
| Аргумент | Описание | Пример |
|---|---|---|
| `--data` | Путь к mikado_data.xlsx [обязательный] | `--data "путь/mikado_data.xlsx"` |
| `--rows` | Строки Excel для обработки | `--rows 2-5` или `--rows 3` или `--rows 2-4,7` |
| `--logo` | Путь к PNG логотипу | `--logo "C:/путь/logo.png"` |
| `--images` | Папка с исходными фото | по умолчанию: `images/` рядом с data |
| `--out` | Папка для результатов | по умолчанию: `photos_ozon/` рядом с data |
| `--size` | Размер фото в пикселях | `--size 1000` (по умолчанию) |

### Зависимости (только две):
```
uv run --with pillow --with openpyxl ...
```
rembg **не используется** — фон и вотермарки поставщика сохраняются намеренно.

## ЧТО ДЕЛАЕТ СКРИПТ ПО ШАГАМ

1. Читает `mikado_data.xlsx` — берёт колонки: Код, Наименование, Бренд, Параметры
2. Фильтрует по `--rows` (номера строк Excel, начиная с 2)
3. Находит фото в папке `images/` по артикулу (например `f-a22025_1.jpg`)
4. Для каждого фото:
   - Обрезает нижнюю полосу с артикулом поставщика (`crop_article_strip`)
   - Масштабирует по ширине 1000px (`_fit_to_width`)
   - Сохраняет `_main.jpg` (без наложений)
   - Накладывает полупрозрачные подложки сверху/снизу → рисует логотип, название, параметры
   - Сохраняет `_info.jpg`
5. Результаты в `photos_ozon/ДАТА_ВРЕМЯ/`

## КАК РАБОТАЕТ ОБРЕЗКА ПОЛОСЫ АРТИКУЛА

Исходные фото FENOX (поставщик mikado) имеют в нижней части светлую полосу с артикулом
(например "A22 025"). Фон полосы заметно светлее основной области (~218 vs ~197 по яркости).

Функция `crop_article_strip()`:
- Сканирует строки начиная с 65% высоты изображения
- Ищет строку где средняя яркость резко прыгает вверх (>8 пунктов за одну строку)
- Обрезает изображение по этой строке
- Протестировано: 538×400 → 538×359 (убирает ~41px полосы)

## ТИПИЧНЫЕ ПРОБЛЕМЫ

| Проблема | Решение |
|---|---|
| Полоса артикула не обрезается | Порог скачка яркости (сейчас >8) — подобрать под конкретные изображения |
| Логотип не виден | Проверить путь к файлу, он должен быть PNG |
| Параметры не отображаются | Проверить колонку "Параметры" в mikado_data.xlsx |
| PermissionError на xlsx | Закрыть файл в Excel |
