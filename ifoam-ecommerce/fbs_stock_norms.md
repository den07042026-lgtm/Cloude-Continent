# Промт: Нормы запасов FBS по складам продавца

## Назначение

Модуль рассчитывает **оптимальную норму запаса** для каждого товара на каждом складе продавца
(FBS / rFBS) и сравнивает её с фактическим свободным остатком.
Результат — таблица с рекомендациями к пополнению или выявлением излишков, а также три
готовых файла загрузки остатков в Ozon.

## Что делает скрипт

| Шаг | Действие | Источник |
|-----|----------|---------|
| 1 | Получает названия складов продавца | `POST /v1/warehouse/list` |
| 2 | Загружает прогноз продаж по складам продавца | `ozon_forecast_detailed_adaptive.csv` |
| 3 | Строит 14-месячную историю `forecast_quantity` по (article_id × склад) | — |
| 4 | Рассчитывает норму запаса по формуле safety-stock | — |
| 5 | Сравнивает норму с фактом из `stocks_fbs_latest.parquet` | — |
| 6 | Создаёт аналитический Excel + 3 файла загрузки остатков в Ozon | файловая система |

## Формула нормы запаса

```
mean      = среднее forecast_quantity за 14 месяцев
std       = STDEV.P по тем же данным
safety_days = Z(95%) × std × √lead_time × √30.44 / mean
total_days  = lead_time + safety_days          (lead_time = 14 дней)
optimal_days = min(total_days, shelf_life − 120)
norm_qty    = ⌈optimal_days × mean / 30.44⌉
            → округление вверх до кратного "В коробке"
```

`Z(95%) ≈ 1.645` — уровень сервиса 95%.
Норма **всегда кратна** количеству товара в коробе.

## Схема аналитической таблицы

| Колонка | Тип | Описание |
|---------|-----|---------|
| `Название склада` | str | Из API /v1/warehouse/list |
| `article_id` | str | Первые 6 цифр offer_id (ключ товара) |
| `offer_id` | str | Полный артикул Ozon; если пуст — из products_reference |
| `warehouse_id` | str | ID склада продавца |
| `Свободный остаток` | int | Фактически доступно на складе |
| `В наличии` | int | Всего на складе (вкл. зарезервированное) |
| `Зарезервировано` | int | Зарезервировано покупателями |
| `Среднее_в_мес` | float | Среднее forecast_quantity за 14 месяцев |
| `Std_в_мес` | float | Стандартное отклонение (STDEV.P) |
| `Норма_запаса_шт` | int | Расчётная норма, кратная коробу |
| `К пополнению` | int | max(0, Норма − Свободный остаток); **красные строки** |
| `Излишек` | int | max(0, Свободный остаток − Норма); **жёлтые строки** |
| `В коробке` | int | Кратность пополнения (из каталога) |
| `История_месяцев` | int | Глубина использованной истории |

## Цветовое кодирование Excel

| Цвет | Условие | Действие |
|------|---------|---------|
| 🔴 Красный | `К пополнению > 0` | Требуется пополнение |
| 🟡 Жёлтый | `Излишек > 0` | Избыточный запас |
| Без заливки | Норма = Остаток | Запас в норме |

## Варианты файлов загрузки

Файлы загрузки (`fbs_upload_*.xlsx`) заполняются по шаблону Ozon и содержат листы
**«Остатки на складе»** с колонками:

| Колонка в шаблоне | Вариант `_fact` | Вариант `_norm` | Вариант `_mixed` |
|-------------------|----------------|----------------|-----------------|
| Название склада (идентификатор) | из шаблона | из шаблона | из шаблона |
| Артикул | offer_id | offer_id | offer_id |
| Название товара | — | — | — |
| Доступно на складе, шт | Свободный остаток | Норма_запаса_шт | Если факт=0 → норма, иначе факт |

**Когда использовать каждый вариант:**
- `_fact` — «что есть сейчас» для сверки
- `_norm` — загрузить норму как целевой остаток
- `_mixed` — практический вариант: не трогаем товары с остатком, для нулей устанавливаем норму
