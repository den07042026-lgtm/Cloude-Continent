```python
import pandas as pd

result_file = r"D:\Аналитика\Система_v2\data\output\unit_economics\unit_economics_result.xlsx"

print(f"Расход на рекламу: {df_реклама['Расход_реклама'].sum():,.0f} руб.")
```

### 3.2 Сравнение FBO vs FBS

```python
# Фильтруем строки "Реклама" из df_товары (они добавляются при distribute рекламы)
scheme = df_товары[~df_товары['Схема работы'].isin(['Реклама'])].groupby('Схема работы').agg(

    Продажи       = ('Продажи',        'sum'),
    Комиссии      = ('Комиссии',       'sum'),
```

## Шаг 4 — Формулировка выводов

**Общая картина (из df_итого):**
- Какова общая доля прибыли с учётом рекламы?
- Что занимает наибольшую долю в расходах: комиссии, логистика, себестоимость или реклама?
- Каков расход на рекламу относительно продаж (ДРР общий)?

**Помесячная динамика (df_месяцы):**
- Есть ли рост или падение продаж по месяцам?
- В каком месяце лучшая и худшая рентабельность?
- Есть ли сезонность?

**По товарам (top10_profit):**
- Какие товары приносят наибольшую прибыль?
- Есть ли товары с отрицательной или нулевой прибылью?
- Какой разброс в рентабельности между товарами?

**За 30 дней vs за 3 месяца:**
- Как изменилась рентабельность топ-товаров в последние 30 дней vs 3 месяца?
- Есть ли товары, которые резко ухудшили или улучшили показатели?

**Реклама (ads_by_sku):**
- Какие товары имеют наибольший рекламный расход?
- У каких товаров ДРР выше нормы (> 20%)?
- Есть ли товары с рекламой, но без продаж?

**FBO vs FBS (scheme):**
- Какая схема выгоднее по доле прибыли?
- Чем объясняется разница в комиссиях и логистике?

**Ценообразование (df_ценообраз):**
- Сколько товаров требуют изменения цены (Дней_изменения > 0)?
- Сколько товаров уже помечено «+» (автоматически и вручную)?
- У каких товаров наибольшее расхождение между ценой и полной себестоимостью?

**Качество данных:**
- Сколько артикулов без себестоимости?
- Сколько строк с нулевой логистикой?
- Сколько комиссий подставлено из истории?

Каждый блок — 2–4 предложения с цифрами.

## Контекст о данных

- Данные: заказы Ozon (FBO + FBS) все статусы + рекламные расходы, март 2025 — по сегодня
- **cancelled** — хранятся в листе "Детализация" для справки, но **исключены** из ИТОГО и листа "По месяцам"
- **Продажи** = `price` × `quantity`
- **Комиссии** = `|fin_commission_amount|`; если = 0 — подставляется среднее за 7/14/28 дней из delivered-заказов того же артикула и кластера (`fin_cluster_to`)
- **Логистика**:
  - до 06.04.2026 — `fbo/fbs_trans_max` × `quantity` (справочник `ozon_products_20260325_154822`)
  - с 06.04.2026  — `(fbo/fbs_trans_min + fbo/fbs_trans_max) / 2` × `quantity` (последний актуальный справочник)
- **Справочник товаров** обновляется автоматически при каждом запуске скрипта (`ozon_products.py`)
- **Себестоимость** = из справочника `Себестоимость наборов ОЗОН.xlsx`, ключ — первые 6 символов артикула (`Артикул_6`)
- **Расход_реклама** = расход по Ozon Performance API (`money_spent` с НДС, руб.); в листе "По товарам" распределяется пропорционально количеству заказов по `Артикул_6`
- **Прибыль** (по товарам) = Продажи − Комиссии − Логистика − Себестоимость − Расход_реклама
- **Прибыль** (ИТОГО) = сумма по заказам − общий расход на рекламу
- **ДРР** = Расход_реклама / Продажи по тому же offer_id
- **Листы "За 30 дней" и "За 3 месяца"** — те же колонки что "По товарам", срезы по дате от максимальной даты заказов
- **fin_cluster_to** — кластер назначения; влияет на комиссию и логистику
- Строки без себестоимости: добавить артикулы в `Себестоимость наборов ОЗОН.xlsx`
- Рекламные данные собираются скриптом `modules/01_collect/ozon_performance.py`

## Лист «Ценообразование» — структура и логика

Формируется на основе данных за последние 30 дней. Колонки:

| Колонка | Описание |
|---------|----------|
| `offer_id` | Артикул товара |
| `Участвует` | «+» — товар участвует в изменении цены (авто или вручную) |
| `Категория` | Категория товара |
| `Количество` | Продано штук за 30 дней |
| `Продажи` | Выручка за 30 дней, руб. |
| `Цена_продавца` | Цена из последнего известного заказа |
| `Полная_себестоимость` | (Себестоимость + Комиссии + Логистика + Реклама) / Количество |
| `Маржа` | Прибыль / Продажи |
| `Изменение_цены` | Целевое изменение цены (руб.) |
| `Дней_изменения` | Количество дней до достижения целевой цены (шаг 2%/день, округление вверх) |
| `День_1..N` | Цена на каждый день стратегии |

**Автопометка «+»:**
- Топ-3 товара по продажам с отрицательной маржой
- Товары с маржой > 15% и количеством < 50 шт

**Применение цен:**
```bash
python modules/01_collect/ozon_update_prices.py           # применить День_1
python modules/01_collect/ozon_update_prices.py --day 2   # применить конкретный день
python modules/01_collect/ozon_update_prices.py --dry-run # только просмотр
```
При ошибке `price_less_than_min_auto_price` скрипт автоматически снижает минимальную цену товара, затем повторяет обновление.
