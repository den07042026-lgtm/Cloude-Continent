# Промпт: Анализ остатков FBS / rFBS / FBP (Ozon)

## Назначение

Модуль собирает актуальные складские остатки по всем товарам продавца по схемам
**FBS** (со своего склада), **rFBS** (со склада партнёра) и **FBP** (Ozon Premium / Pickup).
Результат — плоская таблица с количеством в наличии и зарезервированным для каждого товара
по каждой схеме и складу.

> Для остатков по схеме **FBO** используйте модуль `ozon_stocks.py`
> (API: `/v1/analytics/stocks`).

---

## Запуск

```bash
# из корня Система_v2
python -m modules.01_collect.ozon_stocks_fbs
```

Или напрямую:

```bash
python modules/01_collect/ozon_stocks_fbs.py
```

---

## Что делает скрипт

| Шаг | Действие | API / источник |
|-----|----------|----------------|
| 1 | Получает все `offer_id` продавца (пагинация по `last_id`) | `POST /v3/product/list` |
| 2 | Запрашивает остатки FBS/rFBS/FBP батчами по 1 000 offer_id; внутри батча — cursor-пагинация | `POST /v2/product/info/stocks` |
| 3 | Разворачивает вложенный массив `stocks[]` в плоские строки; строки с `type=fbo` пропускаются | — |
| 4 | Сохраняет результаты в parquet / csv / xlsx | файловая система |

| Файл | Описание |

| `~/Documents/stock-update-template.xlsx` | Сводная таблица для ручной работы (лист «Остатки FBS-rFBS-FBP») |

## Схема таблицы

| Колонка | Тип | Описание |
|---------|-----|----------|
| `offer_id` | str | Артикул продавца |
| `product_id` | int | ID товара в Ozon |
| `Схема` | str | FBS / rFBS / FBP |
| `SKU` | int | SKU склада |
| `В наличии` | int | Количество единиц на складе |
| `Зарезервировано` | int | Зарезервировано покупателями |
| `Тип отгрузки` | str | `SHIPMENT_TYPE_GENERAL` и др. |
| `Склады (IDs)` | str | ID складов через запятую |
| `snapshot_date` | date | Дата снимка |

Ты — аналитик маркетплейса Ozon. У тебя есть таблица остатков товаров
по схемам FBS, rFBS и FBP со следующими колонками:
`offer_id`, `Схема`, `SKU`, `В наличии`, `Зарезервировано`, `snapshot_date`.

Задача: {ЗАДАЧА}

Данные:
```
{ВСТАВЬ_CSV_ИЛИ_ТАБЛИЦУ}
```

Ответь структурированно:

1. **Сводка по схемам** — суммарно «В наличии» и «Зарезервировано» по каждой схеме (FBS / rFBS / FBP).
2. **Критически низкие остатки** — товары, у которых `В наличии` ≤ 5 единиц.
