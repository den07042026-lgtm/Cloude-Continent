# Промт: Генерация справочника комплектаций (BOM)

## Задача
Создать справочник комплектаций (Bill of Materials) для товаров ifoam.
Для каждого базового артикула из прайс-листа генерируется набор записей,
описывающих из каких компонентов состоит готовый продукт по каждому каналу продаж.

## Входные данные
- `data/reference/product_catalog.parquet` — каталог товаров (колонки: Артикул, Категория, Объём, В коробке)
- Категории: `Автохимия и автокосметика`, `Бытовая химия`, `Клининг`

## Структура выходной таблицы (BOM)

| Колонка | Описание |
|---------|---------|
| `Артикул_ГП` | Артикул готового продукта (ge_/re_/oz_/wb_ + базовый артикул) |
| `Артикул_комплектующего` | Артикул компонента |
| `Количество_на_единицу_ГП` | Сколько единиц компонента на 1 единицу ГП |
| `Комментарий` | Пояснение |

## Каналы продаж (4 варианта на каждый артикул)

| Префикс | Канал | Тип короба |
|---------|-------|-----------|
| `ge_` | Общий (оптовый) | Оптовый `mv_` |
| `re_` | Сетевые клиенты | Оптовый `mv_` |
| `oz_` | Ozon | Индивидуальный `mvo_`, qty=1 |
| `wb_` | Wildberries | Индивидуальный `mvo_`, qty=1 |

## Строки BOM на каждый базовый артикул (12 строк)

```
ge_ART  → ART,         qty = 1
ge_ART  → mv_VOL,      qty = 1 / (кол-во в коробке)
re_ART  → ART,         qty = 1
re_ART  → mv_VOL,      qty = 1 / (кол-во в коробке)
oz_ART  → ART,         qty = 1
oz_ART  → mvo_VOL,     qty = 1
wb_ART  → ART,         qty = 1
wb_ART  → mvo_VOL,     qty = 1
ART     → tg_f_ART,    qty = 1                      (фронт-этикетка)
ART     → tg_b_ART,    qty = 1                      (бэк-этикетка)
ART     → fl_VOL,      qty = объём в литрах          (флакон/канистра)
ART     → fc_XXXX00,   qty = объём в литрах          (химическая формула-наполнитель)
```

## Система именования компонентов: prefix_XXXXXX

Все придуманные артикулы: **префикс + 6-значный код**.

### Химическая формула-наполнитель: `fc_`

Артикул формулы = **`fc_` + первые 4 цифры базового артикула + `00`**

```
Артикул 131101, 131105, 131122  →  fc_131100  (CHEAP, одна формула для всех объёмов)
Артикул 208101, 208105          →  fc_208100  (BUG-OFF концентрат)
Артикул 208250                  →  fc_208200  (BUG-OFF готовый — другая формула: разбавленная)
```

Логика: все SKU одного продукта (разные объёмы) используют **одну формулу**.
Концентрат и готовый вариант (триггер) — **разные формулы** (разная степень разбавления).

Количество `fc_` в BOM = объём в литрах (аналогично флакону).

### Правило формирования 6-значного кода объёма

```
1 кг   → 001000   (1000 мл)
5 кг   → 005000   (5000 мл)
5,5 кг → 005500   (5500 мл)
6 кг   → 006000   (6000 мл)
500 мл → 000500
20+ кг → 020000   (один тип канистры для 20, 21, 22, 23, 24, 25, 26 кг)
```

**Исключение:** артикул `206250` (Motor Shine готовый, 500 мл) —
отдельный флакон и отдельный короб. Код = сам артикул: `fl_206250`, `mv_206250`, `mvo_206250`.

## Два варианта упаковки для oz_ / wb_

**Вариант А — индивидуальный короб** (12 строк на SKU):
```
oz_АРТ → mvo_ОБЪ,  qty = 1
wb_АРТ → mvo_ОБЪ,  qty = 1
```

**Вариант Б — плёночная упаковка** (13 строк на SKU):
```
oz_АРТ → pb_ОБЪ,   qty = 1   (пузырчатая плёнка)
oz_АРТ → sl_ОБЪ,   qty = 1   (рукав из плёнки)
wb_АРТ → pb_ОБЪ,   qty = 1
wb_АРТ → sl_ОБЪ,   qty = 1
```

Тип упаковки определяется для каждого товара на основе реестра заказчика.
oz_ и wb_ всегда используют одинаковую упаковку (mvo_ или pb_+sl_).

## Особые правила

1. **Этикетки** — всегда уникальны для базового артикула, одинаковы для всех каналов.
2. **Флаконы** — одинаковы для всех продуктов одного объёма (разная плотность = разный вес, но тот же флакон).
3. **Канистры 20–26 кг** — один тип физической ёмкости (`fl_020000`, `mv_020000`, `mvo_020000`), количество флакона в BOM = фактический объём в литрах.
4. **oz_ и wb_** — используют одинаковую упаковку: либо оба `mvo_` (вариант А), либо оба `pb_` + `sl_` (вариант Б).
5. **206250** — единственное исключение с отдельным флаконом и коробом.

## Количество флакона в BOM

Количество = объём в литрах:
- 1 кг → 1.0
- 5 кг → 5.0
- 5,5 кг → 5.5
- 6 кг → 6.0
- 500 мл → 0.5
- 22 кг → 22.0 (и т.д. для каждого конкретного артикула)

## Количество короба для ge/re каналов

```
qty = 1 / (кол-во в коробке)

1 шт в коробе  → qty = 1.0
4 шт в коробе  → qty = 0.25
15 шт в коробе → qty = 0.066667
20 шт в коробе → qty = 0.05
```

## Расширение на другие категории

При добавлении Бытовой химии или Клининга:
- Та же логика, но могут появиться новые объёмы (например, 750 мл, 1000 мл, 1500 мл, 5 л)
- Новые коды: `fl_000750`, `fl_001000` (может совпасть), `fl_001500`, `fl_005000`
- Для бытовой химии добавляются флаконы с дозаторами — требуют отдельных кодов
- Выходные файлы: `bom_bytovaya.parquet`, `bom_klining.parquet`
- Итоговый объединённый BOM: `bom_all.parquet`
